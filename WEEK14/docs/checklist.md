## Before Lecture/Seminar

### Technical Preparation
- [ ] Demonstration VM prepared and tested
- [ ] Starterkit functional (ran `make run-demo` successfully)
- [ ] Backup: pcap captures prepared (if live demo fails)
- [ ] Projector/screen functional

### Materials
- [ ] Slides/notes for review
- [ ] Project evaluation rubric (printed or digital)
- [ ] Team list and presentation order
- [ ] Defence questions prepared

### Administrative
- [ ] List of students present
- [ ] Project grading form

---

## During the Lecture

### Review (45 min)
- [ ] Present "big picture" (complete communication model)
- [ ] Diagnostic checklist (on board/slide)
- [ ] Mini-demo 1: TCP handshake in pcap
- [ ] Mini-demo 2: Connection refused vs timeout
- [ ] Check questions (2-3 throughout)

### Connection to Project (15 min)
- [ ] What we expect at presentation
- [ ] Examples of defence questions
- [ ] Rubric explained

### Final Q&A (15 min)
- [ ] Questions about exam
- [ ] Clarifications about project

---

## During the Seminar

### Setup (10 min)
- [ ] Verify that students have access to environment
- [ ] Establish presentation order
- [ ] Explain rules (time, Q&A)

### Presentations (10-12 min/team)
- [ ] Keep time
- [ ] Grade on rubric
- [ ] Ask 2-3 defence questions
- [ ] Short feedback (optional, at the end)

### Closing
- [ ] Announce results (if ready)
- [ ] Remind: exam access
- [ ] Useful links for preparation

---

## After the Seminar

### Evaluation
- [ ] Finalise grades on rubric
- [ ] Enter into register
- [ ] Mark students who did NOT present

### Follow-up
- [ ] Email with results (optional)
- [ ] Resources for exam preparation
- [ ] Feedback for kit improvement

---

## Common Pitfalls and Solutions

| Pitfall | Solution |
|---------|----------|
| Demo does not start | Use prepared captures |
| Students do not have VM | Demonstration on staff VM |
| Questions too difficult | Adjust to actual level |
| Time exceeded | Stop politely, grade partially |
| Port conflicts | `make clean` before each demo |
