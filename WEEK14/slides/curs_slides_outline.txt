SECTION B: LAYERED MODEL - SYNTHESIS (15 min, 6 slides)
--------------------------------------------------------------------------------

SLIDE 4: The 7 layers - Overview
- Physical → Data Link → Network → Transport → Session → Presentation → Application
- Practical focus: Data Link, Network, Transport, Application
- Visual: Vertical stack with protocol examples per layer

SLIDE 5: Data Link Layer
- Role: Hop-to-hop communication, Ethernet frames
- Addressing: MAC (48 bits, format: aa:bb:cc:dd:ee:ff)
- Key concepts: Switch, broadcast domain, ARP
- Control question: "How does a host learn the neighbour's MAC?"

SLIDE 6: Network Layer
- Role: End-to-end routing, IP packets
- Addressing: IPv4 (32 bits), CIDR subnetting
- Key concepts: Router, TTL, fragmentation, default gateway
- Control question: "What happens when TTL reaches 0?"

SLIDE 7: Transport Layer
- TCP vs UDP - synthetic comparison
  - TCP: connection, reliability, flow control, congestion control
  - UDP: connectionless, best-effort, minimal overhead
- Ports: <1024 privileged, >49151 ephemeral
- Control question: "When do you choose UDP instead of TCP?"

SLIDE 8: Application Layer
- HTTP/HTTPS: request-response, methods, status codes
- DNS: name resolution, hierarchy, caching
- SMTP/IMAP: email flow (week 13)
- Visual: Simplified HTTP GET flow diagram

SLIDE 9: Encapsulation - The complete picture
- Data → Segment → Packet → Frame → Bits
- Headers added at each layer
- Decapsulation at destination (reverse order)
- Visual: Encapsulation animation with colours per layer

================================================================================
SECTION C: ADDRESSING AND IDENTIFICATION (10 min, 4 slides)
--------------------------------------------------------------------------------

SLIDE 10: Address types - Complete catalogue
- MAC: hardware identification, local to segment
- IP: logical identification, globally routable
- Port: process/service identification
- Visual: 3-column table with characteristics

SLIDE 11: Subnetting and CIDR - Practical review
- CIDR notation: IP/prefix (e.g.: 192.168.1.0/24)
- Calculating available addresses: 2^(32-prefix) - 2
- Network address vs Broadcast address
- Solved example: 10.0.0.0/22 → how many hosts?

SLIDE 12: NAT and private addresses
- Private classes: 10.x, 172.16-31.x, 192.168.x
- NAT: address translation, IPv4 conservation
- Port forwarding: exposing internal services
- Visual: NAT diagram with translation table

SLIDE 13: DNS - From name to IP
- Hierarchy: Root → TLD → Authoritative
- Query types: A, AAAA, MX, CNAME, TXT
- Caching and TTL: performance vs update
- Control question: "Why low TTL for failover?"

================================================================================
SECTION D: TCP IN DETAIL (10 min, 4 slides)
--------------------------------------------------------------------------------

SLIDE 14: Three-Way Handshake
- SYN → SYN-ACK → ACK
- Sequence numbering: ISN (Initial Sequence Number)
- Why 3 steps? Bidirectional synchronisation
- Visual: Sequence diagram with timestamps

SLIDE 15: Data transmission
- Sequence number, Acknowledgment number
- Sliding window: transmission efficiency
- Retransmission on timeout or duplicate ACK
- Visual: Packet timeline with ACKs

SLIDE 16: Flow Control and Congestion Control
- Flow control: receiver window, flood prevention
- Congestion control: slow start, congestion avoidance
- cwnd vs rwnd: which is the limit?
- Visual: Graph of cwnd evolution over time

SLIDE 17: Connection termination
- FIN → ACK → FIN → ACK (four-way)
- TIME_WAIT state: why 2×MSL?
- RST: abrupt termination
- Control question: "What problems occur without TIME_WAIT?"

================================================================================
SECTION E: DIAGNOSTIC CHECKLIST (15 min, 5 slides)
--------------------------------------------------------------------------------

SLIDE 18: Systematic methodology
- Why checklist? Avoid assumptions, save time
- Bottom-up vs Top-down: when to use each
- Documentation: note what you have verified
- Visual: Decision flowchart

SLIDE 19: Steps 1-2: Local verification
- Step 1: IP and configuration (`ip addr`, `ip route`)
- Step 2: Local connectivity (`ping gateway`)
- What to look for: valid IP, correct gateway, link up
- Typical mistakes: DHCP not working, cable disconnected

SLIDE 20: Steps 3-4: External connectivity
- Step 3: DNS functional (`nslookup`, `dig`)
- Step 4: Internet connectivity (`ping 8.8.8.8`)
- What to look for: resolution OK, packets arriving
- Typical mistakes: wrong DNS, firewall blocking

SLIDE 21: Steps 5-6: Application and analysis
- Step 5: Service port (`netstat -tlnp`, `ss`)
- Step 6: Traffic capture (`tcpdump`, `tshark`)
- What to look for: service listening, correct packets
- Typical mistakes: wrong port, local firewall

SLIDE 22: Key tools
- Table: Tool → What it verifies → Typical command
  - ping: ICMP connectivity
  - traceroute: packet path
  - netstat/ss: open ports
  - tcpdump/tshark: packet contents
  - curl/nc: application test
- Visual: Icons for each tool
